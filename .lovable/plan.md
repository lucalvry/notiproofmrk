

# Koray's Semantic SEO Audit — Gaps & Remediation Plan

## What's Already Done Well

- **SEOHead component** on every page with title, meta description, canonical, OG/Twitter tags
- **JSON-LD structured data**: Organization, WebSite, SoftwareApplication, FAQPage, Article, BreadcrumbList schemas all present
- **BreadcrumbList schema** auto-generated via `Breadcrumbs.tsx` with visual breadcrumbs
- **Pillar-and-cluster content model**: Social Proof pillar page with 6 cluster articles, dense internal contextual linking between them
- **Heading hierarchy**: H1 per page, H2 for sections, H3 for sub-items — generally correct
- **Footer with dense internal links** across all content clusters
- **Canonical URLs** set on all commercial and content pages
- **FAQ sections with FAQPage schema** on product, use-case, integration, and pricing pages
- **robots.txt** properly configured

---

## Gaps Found

### 1. Missing `sitemap.xml` (Critical)
`robots.txt` references `https://notiproof.com/sitemap.xml` but **no sitemap file exists**. Since this is a client-side SPA, we need a static `public/sitemap.xml` listing all 44+ URLs with `<lastmod>` dates.

**Fix**: Create `public/sitemap.xml` with all route URLs.

### 2. No Internal Contextual Links on Homepage (High)
Koray emphasizes that **every page should link contextually to related topical pages** using descriptive anchor text (not just "Learn more"). The homepage currently links to use cases and pricing but has zero contextual links to resource articles, the social proof pillar, or product sub-pages from within paragraph/body text. Feature cards and use-case cards link out, but there's no prose-level contextual linking.

**Fix**: Add a short "Learn More About Social Proof" section on the homepage with 3-4 contextual paragraph-style links to the pillar page and key cluster articles.

### 3. Missing Contextual Internal Links on Product/Use-Case/Integration Pages (High)
Product pages (`ProductFeaturePage`) link to related products via buttons but lack **in-body contextual links** — the kind Koray considers essential ("anchor text within flowing prose"). The benefits and descriptions are plain text with no links to related resources, pillar pages, or other product pages.

**Fix**: Update template components to accept optional `bodyLinks` or add hardcoded contextual sentences with internal links in each product/use-case page's description or benefits sections. At minimum, add a "Related Resources" prose section to `ProductFeaturePage` and `UseCaseTemplate` that links to relevant resource articles.

### 4. Resource Category Hubs Have No Body Content (Medium)
The category hub pages (`ReviewsHub`, `ConversionAnalyticsHub`, `WebsiteTrustHub`, `TestimonialsResourceHub`) are generated by `makeHub()` — they render only a title and a grid of link cards. Koray's approach requires hubs to have **introductory topical content** (2-3 paragraphs explaining the topic) to establish entity relevance, not just a list of links.

**Fix**: Expand `makeHub()` or convert each hub to a standalone component with an intro paragraph, topical context, and then the article grid.

### 5. No `<article>` Semantic HTML on Resource Pages (Medium)
Resource articles use `ResourceArticle` component but render content inside generic `<div>` tags. Using `<article>`, `<time>`, and `<address>` elements improves semantic signals for search engines.

**Fix**: Wrap resource article content in `<article>` with `<time datetime="...">` for publish dates.

### 6. Missing Entity Optimization / "About" Entity Signals (Medium)
Koray stresses defining your brand as a **named entity** that Google can associate with a topic. The `Organization` schema exists but is minimal — missing `foundingDate`, `founders`, `numberOfEmployees`, `knowsAbout` (topical expertise), and `areaServed`.

**Fix**: Expand the Organization schema in `index.html` and `Index.tsx` with entity-enriching properties.

### 7. No Topical Map Cross-Linking Between Clusters (Medium)
The social proof cluster is well-linked internally, but there's no linking **between clusters** (e.g., Reviews hub linking to Social Proof pillar, Testimonials resources linking to product pages). Koray's model requires **cross-cluster contextual bridges**.

**Fix**: Add 2-3 contextual cross-links in each resource article to related product pages and other cluster hubs.

### 8. Auth/Legal Pages Missing `noindex` (Low)
Login, Signup, Forgot Password, and Reset Password pages have SEO metadata but should be `noindex` to avoid thin/duplicate content in the index.

**Fix**: Add `noindex={true}` to SEOHead on auth pages.

### 9. Social Links Missing `target="_blank"` (Low)
Footer social links (Twitter, LinkedIn) lack `target="_blank"`, meaning they navigate away from the site.

**Fix**: Add `target="_blank"` to external social links in the Footer.
